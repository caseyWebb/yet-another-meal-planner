## Why

Disjunctive surface forms — "X or Y" — are being captured as CONCRETE canonical identities. A disjunction is not a product identity; it is a satisfaction constraint (either X or Y fulfills the line). The DSL already models this correctly with ABSTRACT concept nodes (`concrete=0`) plus `membership` satisfies-edges (`hot sauces (various)`, `korean ramen toppings (seafood)` in production), but no behavior enforces it: capture happily mints `white or yellow onion` as a buyable product, no pass ever re-litigates an existing node's SHAPE (the audits re-check decisions, not identity modeling), and the matcher sends the disjunctive phrase to Kroger verbatim (where the token `or` also poisons substring relevance scoring).

Production holds exactly three disjunctive families, all `source='auto'`, all with live grocery/pantry rows: `white or yellow onion`, `anaheim or cubanelle peppers`, and the `serrano or jalapeño peppers` family — the worst case, where the bare disjunction node was re-confirm-merged INTO its own `::form-diced` child. That inversion is also driving a live per-tick churn loop: the edge-audit pre-pass deletes the structural edge `serrano or jalapeño peppers::form-diced -[general]→ serrano or jalapeño peppers` as a representative-resolved self-loop, then the structural guarantee re-inserts it for the surviving child — one `edge_drop` + one `edge_restore` logged every ~5-minute tick, ongoing (16 drops / 14 restores and counting). The shape sweep in this change is what quiesces it.

Per the working mode, production converges through the pipeline: the three families are this change's acceptance fixtures and the sweep must fix them — no manual data edits.

## What Changes

- **Capture-time disjunction disposal (deterministic, no LLM):** a queued term matching the disjunctive pattern (the token ` or `, including comma lists) is never confirmed as a concrete identity. After the lexical fast path and before the floor/classifier, plain code mints it as an abstract concept node (`concrete=0`, id = the cleaned term, `search_term` = the first disjunct after head-noun distribution, embedding stored), aliases the surface form to it, and commits through the existing `commitResolution`. The classifier is never called for these terms; belt-and-braces, the confirm prompt gains a "never mint a disjunctive identity" rule and a classifier-proposed NOVEL canonical matching the pattern is rejected to the verbatim fallback. The alias re-audit applies the same deterministic disposal to a disjunctive-pattern variant.
- **Disjunction membership reconcile (structural-guarantee sibling):** a deterministic per-tick sub-pass of the capture job splits every surviving abstract disjunction node's id into disjunct terms (head-distribution rule), resolves each through the full front door (alias + representative), inserts missing `membership` edges member → concept born-stamped, and enqueues unresolved disjuncts to `novel_ingredient_terms` (insert-or-ignore) until capture places them. Idempotent, bounded, logged.
- **Retroactive shape sweep (the self-healing part):** the same sub-pass flips every surviving CONCRETE `source='auto'` node matching the pattern to abstract (`concrete=0` + member-phrase `search_term`), folds a `::detail` child of a disjunctive base into the (now abstract) base via the representative pointer — including the re-root shape for the production serrano inversion — and mints a missing abstract base for an orphaned child. Human nodes are never touched. Self-quiescing by selection predicate (flipped/folded rows no longer match). Aliases keep resolving; grocery/pantry rows and `sku_cache` keys converge through the existing reconciles; the serrano structural-edge churn loop ends because the folded child stops surviving.
- **Matcher behavior — scoped down to data, no matcher code change:** an abstract disjunction node always carries a member-phrase `search_term` (first disjunct), so the existing search_term-driven pipeline never sends a disjunctive phrase to Kroger, and the existing tri-state confidence gate (no cache, no `[brands]` key → ambiguous) already surfaces member candidates as options rather than silently auto-picking. Production has zero `sku_cache`/`brand_prefs` rows keyed by disjunctive or concept ids, so no cache cleanup is needed.
- **Concept–concrete merge guards:** the SKU co-resolution pass skips a candidate pair whose survivors differ in `concrete` (a member and its concept sharing a SKU is expected once a concept-keyed cache row exists — not synonym evidence), and the re-confirm pass rejects a `same` merge whose survivor is a concept node — both logged, preventing a recurrence of the serrano-style inversion through the merge paths.
- **General oscillation guard (architect-ratified amendment):** the edge-audit structural guarantee never inserts an edge that is currently a representative-resolved self-loop — it skips whenever `resolve(base) === resolve(detail-node)` — closing the whole base-into-own-child inversion class deterministically, so an inverted family's churn quiesces even before the shape sweep converges its shape.

No schema change: the pattern is derived from the id, quiescence is by selection predicate (no new stamp), the queue/log/edge tables are reused as-is. Challenged and confirmed in design D11.

## Capabilities

### New Capabilities

_None._

### Modified Capabilities

- `ingredient-normalization`: four new requirements — disjunctive terms resolve to satisfaction concepts (capture gate + prompt rule + canonical rejection + re-audit parity); the disjunction membership reconcile; the retroactive disjunction shape sweep (production fixtures, churn quiesce); the concept–concrete merge guard on the re-confirm and co-resolution passes — plus one modified requirement: the structural edge guarantee gains the self-loop insertion guard (the general oscillation fix).
- `ingredient-matching`: **no spec delta.** The matcher contract already reads "search using the canonical id's stored `search_term`"; this change only guarantees what that field contains for disjunction concepts. The concept-node purchase behavior ("members surfaced as options, never buy the concept") stays owned by `ingredient-normalization`'s existing concept requirement, which the search_term + ambiguous-gate composition satisfies.
