## MODIFIED Requirements

### Requirement: Code repository layout

The code repository (`caseyWebb/groceries-agent`) SHALL hold the Worker source under `src/` (including the operator admin panel — a Hono SSR + islands app — under `src/admin/`), the build tooling under `scripts/` (including the plugin builder `scripts/build-plugin.mjs`), the agent persona source `AGENT_INSTRUCTIONS.md`, the committed admin island build under `admin/dist/`, the D1 schema migrations under `migrations/d1/`, the reference docs under `docs/`, the OpenSpec workspace under `openspec/`, and the test suites under `test/` (Worker) and `tests/` (build tooling). Reusable CI workflows that operator data repos call SHALL live under `.github/workflows/`. The code repository SHALL NOT commit a generated `plugin/` bundle or a `.claude-plugin/marketplace.json`; the plugin bundle is generated by the operator's deploy into the operator's data repo (see "One data repository as the deploy control plane and plugin marketplace host" and `agent-plugin-distribution`).

#### Scenario: Worker source and tooling are present

- **WHEN** the code repository is cloned and listed
- **THEN** `src/`, `scripts/` (with `build-plugin.mjs`), `migrations/d1/`, `docs/`, `openspec/`, `test/`, and `tests/` are present along with `AGENT_INSTRUCTIONS.md`, and no committed `plugin/` bundle or `.claude-plugin/marketplace.json` exists in the code repository

#### Scenario: The admin island bundle is committed alongside its source

- **WHEN** the admin panel is built
- **THEN** `admin/dist/` (the island bundles + stylesheet) is generated from `src/admin/client/` (+ `src/admin/styles.css`) and committed; the server-rendered pages are produced by the Worker at request time, not pre-built; the plugin bundle is generated into the operator's data repo by the deploy, not committed in the code repository
