## Why

The home-network `packages/scraper` component was born to do one job — fetch walled recipe pages behind the operator's own subscription session and push the parsed facts back to the Worker. But the same off-cloud, outbound-only helper is the natural home for two more jobs the Worker can't do from the Cloudflare edge: **scanning in-store/loyalty sale prices** a member's logged-in store session gates, and **filling a cart** on a store the Worker has no session for. Those are separate future changes; before any of them lands, the shared piece — the component's name, its wire contract, and the trust discipline that keeps a home box from becoming an authority — has to generalize past "recipe scraper."

This change is the **foundation** of that initiative (the "satellite"). It renames the concept `scraper → satellite`, generalizes the wire contract from a recipe-only batch into a **capability-tagged, observations-only** shape (still carrying recipes and nothing else, functionally identical), and — the substantive heart — writes the trust discipline into the living spec as first-class, capability-agnostic requirements: **the satellite is a sensor, not a judge.** It reports only independently-checkable facts; the Worker re-derives every conclusion. Codifying that now, while `recipe-scrape` is the only capability, means the later scan/order capabilities inherit the discipline instead of re-litigating it.

Nothing user-visible changes: no new capability, no behavior change to recipe ingestion. This is a rename plus a contract restructure plus a spec-level statement of principle.

## What Changes

- **Rename the concept `scraper → satellite`** across the codebase vocabulary: the `packages/scraper/` package (→ `packages/satellite/`, `@grocery-agent/satellite`, CLI `grocery-satellite`), the Worker's conceptual references (comments, `readScraperLiveness`/`ScraperLiveness` internals), the admin-panel labels/routes (Discovery › Scrapers → Satellites, the Ingest Keys blurb), the Playwright page objects, and the reference docs. **The OpenSpec `walled-source-scraper` capability is renamed to `satellite`.**
- **The DB and wire endpoint keep their `ingest_*` names.** The D1 tables (`ingest_keys`, `ingest_candidates`, `ingest_pushes`), their migrations, the endpoint `POST /admin/api/ingest`, and the `INGEST_API_KEY` env var are **not** renamed — renaming deployed DB objects or a wire path is risk for no benefit, and "ingest" is capability-neutral (a satellite ingests observations). See design for the compatibility rationale.
- **Generalize the wire contract into a capability-tagged, observations-only shape (`CONTRACT_VERSION` → `"v2"`).** A satellite declares one or more **capabilities**; `recipe-scrape` is the only one in this change. A push carries a `capability` tag and an array of **observation items** that are a **discriminated union keyed by `kind`** (`{ kind: "recipe", ... }` today), so later item kinds add without breaking v1/v2 consumers. The recipe observation carries the identical functional facts v1 carried.
- **Accept both v1 and v2 during the transition.** The Worker's lenient envelope accepts a v1 recipe batch (`{ source, scraper_version, contract_version, recipes[] }`) and a v2 satellite batch (`{ capability, source, satellite_version, contract_version, observations[] }`), normalizing either into the same internal recipe intake. A producer reporting `contract_version: "v1"` reads as **skewed** against the Worker's current `"v2"` (correct — it nudges an image upgrade), and the existing skew detection keeps working unchanged.
- **Codify the trust discipline as capability-agnostic spec requirements** under the new `satellite` capability: observations-only (no wire field for a derived value the Worker cares about), trusted/untrusted convergence at a raw-observation layer, "trust outputs after validation, never the process," irreversible-actions-stay-human-gated, and strictly-outbound-only. These are written to be inherited by the later scan/order capabilities.

## Capabilities

### New Capabilities
- `satellite`: the off-cloud, outbound-only home-network helper (renamed from `walled-source-scraper`), generalized to be capability-tagged and observations-only. Owns: the component's runtime posture (outbound-only, off-cloud, one machine = one key = many sources), the capability-declaration model (`recipe-scrape` only, clean extension point), the capability-tagged discriminated-union wire contract (`v2`), the **sensor-not-judge** trust requirements (observations-only, raw-observation convergence, trust-outputs-not-process, human-gated irreversible actions), plus the recipe-scrape adapter/fetch/session/CLI mechanics carried over from `walled-source-scraper`.

### Modified Capabilities
- `recipe-ingestion`: the `POST /admin/api/ingest` endpoint accepts the v2 capability-tagged batch (and v1 during transition); the per-satellite liveness/skew derivation is stated against `CONTRACT_VERSION = "v2"` and the reported `satellite_version`.
- `repo-structure`: the code-layout requirement names the `satellite` package (was `scraper`).
- `build-automation`: workspace-aware CI and the tagged container release name the `satellite` package and the `satellite-v*` tag / GHCR satellite image (was `scraper` / `scraper-v*`).
- `operator-admin`: the Config › Ingest Keys editor and the Discovery candidate provenance badge use the `satellite` vocabulary in their operator-facing labels.

- **Package rename:** `packages/scraper/` → `packages/satellite/` (all `src/`, `test/`, `README.md`, `Dockerfile`, `docker-compose.example.yml`, `scraper.example.toml` → `satellite.example.toml`, `package.json` name/description/bin, `vitest.config.ts`, `tsconfig.json`); root `package.json` (description + the `test` filter `@grocery-agent/scraper` → `@grocery-agent/satellite`); `pnpm-workspace.yaml` glob unchanged; `aube-lock.yaml` regenerated for the renamed workspace.

- **Worker code (conceptual/label rename, no DB rename):** `src/ingest.ts`, `src/ingest-db.ts` (`readScraperLiveness`/`ScraperLiveness`/`ScraperRollup` → satellite naming; the `last_scraper_version` **column stays**), `src/discovery-sweep.ts`/`src/discovery-db.ts` (comments), `src/index.ts` (route comment; path unchanged), `src/admin/**` (labels, `pages/scrapers.tsx` → `pages/satellites.tsx`, the Discovery sub-nav + `satellite: <origin>` badge, the Config › Ingest Keys blurb, the Status section, `client/ingest-keys.tsx` copy).

- **CI/release:** the workspace typecheck/test filters and the tagged-release workflow rename `scraper` → `satellite` and `scraper-v*` → `satellite-v*` (GHCR image renamed; old images/tags remain valid).
- **Docs (lockstep):** `docs/ARCHITECTURE.md` (the ingest-arm section + the parser note), `docs/SCHEMAS.md` (the `ingest_keys`/`ingest_candidates` prose + the v2 contract shape; column names unchanged), `docs/SELF_HOSTING.md` (the "satellite" section + `satellite login`/`test` verbs), `docs/TOOLS.md` (confirm unchanged — no MCP tool touched).
