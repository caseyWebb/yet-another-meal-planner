## MODIFIED Requirements

### Requirement: Code repository layout

The code repository (`caseyWebb/groceries-agent`) SHALL be an **`aube` workspace monorepo** (workspaces declared in `pnpm-workspace.yaml`, packages cross-depending via the `workspace:*` protocol) whose packages are the **Worker** (the Cloudflare Worker + its operator admin panel — a Hono SSR + islands app — with its build tooling `scripts/`, D1 migrations `migrations/d1/`, and Worker tests), a shared **contract** package, and the home-network **satellite** package. The shared **contract** package SHALL be **workerd-pure** (no Node-only dependencies) because it is imported by BOTH the Worker (workerd runtime) and the satellite (Node runtime); it SHALL hold the recipe-parse spine (JSON-LD extraction + normalize) and the ingest wire types/validators, so the parse and the payload shape can never drift between the two runtimes. The **satellite** package (Node + a headless-browser dependency) SHALL be isolated in its own workspace so its heavy dependencies never enter the Worker build or deploy. The reference docs SHALL remain under `docs/`, the OpenSpec workspace under `openspec/`, and the agent persona source `AGENT_INSTRUCTIONS.md` at a defined location. Reusable CI workflows that operator data repos call SHALL live under `.github/workflows/`. The code repository SHALL NOT commit a generated plugin bundle, a `.claude-plugin/marketplace.json`, or the built admin island bundle; those are generated by the operator's deploy, not committed.

#### Scenario: The repository is a workspaces monorepo

- **WHEN** the code repository is cloned and listed
- **THEN** it resolves as a workspaces monorepo with a Worker package, a shared workerd-pure contract package, and a satellite package, alongside `docs/`, `openspec/`, and `AGENT_INSTRUCTIONS.md`, and no committed plugin bundle, `.claude-plugin/marketplace.json`, or admin `dist/` bundle

#### Scenario: The shared contract package is workerd-pure

- **WHEN** the contract package is built
- **THEN** it carries no Node-only dependency, so both the Worker (workerd) and the satellite (Node) import the same recipe-parse and ingest wire types

#### Scenario: The satellite's heavy deps stay out of the Worker

- **WHEN** the Worker is built or deployed
- **THEN** the satellite package's browser/Node dependencies are not included — they are isolated to the satellite workspace
