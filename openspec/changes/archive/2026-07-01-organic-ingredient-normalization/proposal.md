## Why

Ingredient normalization today is one exact-match dictionary (`normalizeIngredient` → the D1 `aliases` table), hand-maintained via `update_aliases` "only when the user directs an alias edit." It is the join key under `sku_cache`, `brand_prefs`, `ingredients_key`/`perishable_ingredients`, and grocery-list dedup — **none of which are loaded into chat context**, so the ADR-0001 "read-time LLM handles scallion≈green-onion" argument never reaches them. Two failures follow: (1) surface-form **fragmentation** — "scallions" and "green onions" are different keys, so the group resolves the same SKU twice, a brand preference set for one doesn't apply to the other, and perishable-overlap re-ranking misses the match; (2) the table **goes uncurated** because growth requires a human. Worse, the deterministic quantity-stripper actively *destroys* product qualifiers: `"80/20 ground beef"` → `"ground beef"` (fat ratio deleted), while `"1 lb 80/20 ground beef"` → `"80/20 ground beef"` (survives) — the same ingredient normalizes to two different strings.

This is exactly the revival trigger ADR-0001 named for the deferred ingredient work ("buy scallions while green onions sit in the pantry; duplicate grocery rows; missed perishable overlaps"), and the ADR's chief objection ("curated tables go uncurated") is answered by growing the layer **organically, with no user or operator in the loop** — amortizing one cheap LLM call per novel surface form, permanently.

## What Changes

- **Ingredient identity becomes a directed satisfaction graph over canonical nodes**, not a flat canonical string. A node is named `base` or `base::detail[::detail…]` (a readable label): `"80/20 ground beef"` → `ground-beef::fat-80-20`; `"ground beef"` (unspecified) → `ground-beef` (the "-any" anchor); `"scallions"`/`"green onions"` → `green-onion` (synonym-merged). The deterministic **join key is the full canonical id** (after synonym-merge via a `representative` pointer) — for `sku_cache`, `brand_prefs`, grocery-list dedup, and cross-recipe overlap — **never the bare base**, because same-base nodes may be non-interchangeable varieties (`cheese::cheddar` ≠ `cheese::mozzarella`; a base-equality join would silently fuse them). The base is a readable grouping, the matcher's search-term fallback, and the "-any" anchor.
- **Directional satisfaction is captured as edges, consumed at read-time.** `A → B` means "having A satisfies a request for B," so `satisfies(have, want)` is graph reachability: `chicken::whole → chicken::leg_quarters → chicken::thighs → chicken` (a whole bird satisfies a thighs recipe; a thigh does not satisfy a whole-bird recipe — asymmetric). Concept nodes (`⟨fresh-soft-cheese⟩`, marked distinct from buyable SKUs) collect member edges (`cheese::mozzarella::fresh → ⟨fresh-soft-cheese⟩`) so a generic ingredient enumerates its members. Edges are **open-world hints** for read-time LLM reasoning — a missing or wrong edge degrades to the LLM reasoning from world knowledge, never a hard gate or a wrong purchase. This is the ADR-0001 edge-graph scoped to **factual identity/containment/membership**, still excluding taste-substitution. Deterministic edge-consumers beyond read-time surfacing are evidence-gated (deferred).
- **An organic capture loop grows the layer with no human.** On a normalization miss the hot path returns the cleaned term (identical to today, no regression) and enqueues it. A new scheduled job in the one `scheduled()` handler drains the queue: **embedding proposes** nearest known nodes (cosine), a **cheap classifier disposes** identity with a 3-outcome confirm (SAME → synonym-merge / SPECIALIZATION → new node + a satisfies-edge to its general / NOVEL node) and proposes any directional/membership edges to neighbors, and the resolution is written once and read deterministically forever. Below the cosine floor → novel node, no LLM call. Steady state ≈ 0 LLM calls, exactly like the discovery/classify passes.
- **Conservative-by-construction collapse.** Embedding never collapses on its own (it would fuse `baking-soda`≈`baking-powder`, `heavy-cream`≈`half-and-half`); only the confirm can alias-to-existing, and it is biased to **specialize or mint-novel on any doubt**. Specialization *preserves* the distinction rather than destroying it, so the worst case is a missed shared-base (self-heals on a later tick), never a wrong purchase. Prep qualifiers ("diced", "minced", "softened") strip to base; only qualifiers that change *which SKU you'd buy* specialize.
- **Ids are stable join keys.** Canonical ids are append-only; a late-discovered merge (`scallion` and `green-onion` minted separately before the alias surfaced) uses a **union-find `representative` pointer** resolved transitively — zero cross-table key rewrites. The whole alias+identity map stays small and group-shared, loaded wholesale on the hot path like `readAliases` today (embeddings are cron-only, off the hot-path read).
- **The deterministic quantity-stripper stops eating product qualifiers** — it strips a leading quantity only when a unit follows, leaving ambiguous tokens (`80/20`) for the resolver.
- **Operator override + audit, not operator curation.** Every auto-decision is logged (model, candidates, cosine scores) as an evaluated-set/audit surface; a human override always wins (`source: human` beats `auto`), but the layer never *depends* on a human. `update_aliases` continues to work as a manual, authoritative write into the same store.
- The `aliases` table generalizes into a `variant → id` alias front-door plus an `ingredient_identity` registry (base/qualifier/search_term/representative + a cron-owned embedding), mirroring the `recipes` + `recipe_derived` split. Existing `aliases` rows migrate as base-level ids (no `::`) — backward compatible.

## Capabilities

### New Capabilities
- `ingredient-normalization`: the organic identity layer — the base+qualifier identity model and its opaque-qualifier rule; the novel-term queue; the scheduled capture job (embedding-propose → cheap-classifier-dispose 3-outcome confirm, contract-validated with corrective retry, conservative-collapse bias, prep-vs-product stripping); the `ingredient_alias` + `ingredient_identity` stores; union-find merges and id stability; the audit/evaluated-set log; operator override precedence; steady-state and bounded-per-tick guarantees.

### Modified Capabilities
- `ingredient-matching`: the "Alias-driven normalization" step resolves to a structured canonical id (base + optional qualifier) instead of an exact-alias string; the matcher searches a reconstructed `search_term` for qualified ids; `sku_cache`/`brand_prefs` key on the canonical id; the quantity-stripper no longer discards product qualifiers.
- `shared-corpus`: the `aliases` table becomes the `variant → id` alias front-door alongside a new `ingredient_identity` registry (+ novel-term queue + normalization log); `update_aliases` upserts variants against canonical ids; reads resolve term → id → representative.
- `recipe-facet-derivation`: `ingredients_key`/`perishable_ingredients` normalize to **base-level** ids through the shared resolver, so cross-recipe overlap (waste detection, the perishable-overlap re-rank) joins across surface forms.

## Impact

- **Affected code:** `src/matching.ts` (`normalizeIngredient`/`normalizeIngredientList` → structured resolution + base extraction; tighten `stripLeadingQuantity`), `src/corpus-db.ts` (`readAliases` → resolver map load; new identity/queue/log readers+writers), a new `src/ingredient-normalize.ts` capture job wired into `scheduled()` in `src/index.ts`, `src/write-tools.ts` (`update_aliases` against ids; recipe-write normalization), `src/recipe-classify.ts`/`src/discovery.ts`/`src/tools.ts` (call sites enqueue on miss), reuse of `src/embedding.ts` + the `src/discovery-classify.ts` confirm machinery.
- **New D1 migration:** generalize `aliases` (add `id`/`source`), add `ingredient_identity` (base/qualifier/search_term/representative/embedding), `novel_ingredient_terms` queue, `ingredient_normalization_log`. Backfill: existing `aliases` canonicals become base-level ids; a first cron pass over corpus `ingredients_key` vocabulary bootstraps the registry.
- **New dependencies:** none — Workers AI embedding (`@cf/baai/bge-base-en-v1.5`) and the small classifier are already bound and in use.
- **Budgets:** the capture job rides the internal `env.AI`/D1 bucket (embeddings batched via `embedTexts`; no external subrequests), bounded per tick with a `job_health` record — same discipline as flyer-warm/discovery-sweep.
- **Docs:** `docs/ARCHITECTURE.md` (Kroger matching pipeline step 1 + a new "ingredient normalization" capture section), `docs/SCHEMAS.md` (the new tables + id format), `docs/TOOLS.md` (`update_aliases` id semantics), and an **ADR-0001 amendment** recording that the identity-only revival fired on its named trigger — explicitly *not* the deferred substitution graph.
- **Deferred (explicit non-goals):** the substitution edge-graph (stays read-time LLM per ADR-0001 Phase 0); synchronous hot-path read-through (cron-only first; add only if lag bites); brand-token extraction from ingredient strings; base-level brand-preference cascading to qualified ids.
