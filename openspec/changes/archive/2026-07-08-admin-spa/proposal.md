# Proposal: admin-spa

## Why

P6 — the final phase — of the member web app plan (`docs/plans/web-app.md` §7, ratified 2026-07-07). The operator admin panel is the last surface on the old frontend stack: hono/jsx SSR pages plus per-island bespoke fetch code. The islands audit's evidence stands in the code today: navigation, filters, search, and pagination on every list page are full SSR round-trips; the two busiest triage islands (`client/normalize.tsx`, `client/discovery.tsx`) `location.reload()` after every mutation; in-place refetches elsewhere are fire-and-forget with no loading/error modeling beyond each island's hand-rolled state. The islands model cannot grow a cross-page cache or optimistic writes. Meanwhile P0–P5 built and hardened a complete SPA stack (React 19 + Vite + TanStack Router/Query + `packages/ui`) one directory over. Keeping both stacks is permanent dual maintenance of two component kits, two data-flow idioms, two build pipelines; the plan's verdict is **churn over dual maintenance** — rewrite the panel on the member-app stack, then retire the SSR pages, the islands, and their esbuild/Tailwind-CLI build step at parity.

## What Changes

- **NEW workspace `packages/admin-app`** (`@grocery-agent/admin-app`): the operator admin SPA — React 19 + Vite (Rolldown) + TanStack Router/Query, consuming `packages/ui` (`workspace:*`), building into `assets/admin/` (the admin subtree of the Worker's one merged assets root; each builder cleans only its own subtree). No PWA/service worker — the admin panel is an online operator tool.
- **CHANGE `/admin` serving**: the Worker's Access-gated admin app (Hono) keeps its `/admin/api/*` routes and now serves the SPA shell explicitly for every non-API GET (deep link/refresh included) — `/admin*` is `run_worker_first`, so the Worker must serve the shell itself, unlike the member SPA's asset-fallback path. `/admin/assets/*` serves the hashed bundle; a missing admin static asset still 404s (never the member SPA shell); the two legacy 302 redirects (`/admin/logs/discovery`, `/admin/config/aliases`) stay Worker-side. `wrangler.jsonc` is unchanged (`/admin`, `/admin/*` are already enumerated).
- **NEW typed `/admin/api/*` read routes** (the rewrite's only route growth): the SSR pages' data assembly moves behind ~12 `hc`-typed GETs — the Status aggregate, member detail, the Data explorers (recipes list/search + detail, stores, guidance), insights, usage, the all-jobs run log, discovery candidates, the satellites audit surface, and the normalization page/nodes/audit + reconcile reads. Every existing mutation/preview route survives verbatim; `hc` and the route layer are unchanged in kind. Both transports keep calling the same `src/` functions.
- **CHANGE the panel's data flow**: TanStack Query over `hc<AdminApp>` replaces SSR-embedded props + per-island fetch state. Mutations invalidate queries (no `location.reload()`); filters/search/pagination/tabs become router **search params** at today's exact URLs (deep-linkability preserved); bounded list reads are filtered/paginated client-side, parameterized reads (recipe search, guidance browse) stay server-side.
- **RETIRED at parity**: `src/admin/pages/`, `src/admin/client/`, `src/admin/ui/` (kit/layout/icons/health-dock), `styles.css` (Basecoat), the `injectHealthDock` middleware, and `scripts/build-admin.mjs` (both its esbuild-islands and Tailwind-CLI steps — the worker package drops the `esbuild`/`@tailwindcss/cli`/`tailwindcss`/`basecoat-css` devDeps; no `hono/jsx` remains in the Worker). `aubr build:admin` repoints to the admin app's Vite build; the visual layer becomes the shared shadcn/ui system with the operator theme layered over `packages/ui` tokens, per the committed design bundle (`docs/plans/web-app-design/project/*.jsx`).
- **The Playwright gate survives**: the page objects/specs under `packages/worker/admin/visual/` are ported to drive the SPA (same assertions, same seeded fixtures, same screenshots); `aubr test:admin` stays the blocking `admin-ui` CI job with its sticky screenshot comment. The rewrite is not done until the visual suite passes on the SPA.
- **`src/admin/CLAUDE.md` rewritten** to the new modeling standards: the `Loadable`/`assertNever` discipline mapped onto TanStack Query states, server state never copied into `useState`, URL search params as the shareable-state home, and the shadcn/ui + `packages/ui` styling contract.

## Capabilities

### Modified Capabilities

- `operator-admin`: the panel's architecture requirements — serving, routing, data flow, modeling, visual layer, kit, health indicator — move from SSR + islands to the SPA; area content requirements keep their behavior contracts with rendering-tech wording made SPA-true.
- `operator-data-explorer`: the explorers are served by the typed read routes + SPA views (behavioral contracts unchanged; markdown still rendered Worker-side).
- `admin-ui-testing`: the harness drives the built SPA; landmark/dialog wording updated; the gate, determinism, and screenshot-publishing contracts unchanged.
- `group-insights`: Insights renders in the SPA from one fetched payload; toggles still re-render without a network request.
- `build-automation`: CI/deploy build the admin SPA via Vite (workspace-aware); the esbuild islands step is retired; deploy path filters gain `packages/admin-app/**`.
- `repo-structure`: the workspace roster (worker/contract/satellite/app/ui/admin-app) and the "no committed bundles" rule restated for the merged assets root.

## Impact

- **Code**: new `packages/admin-app/` (routes, screens, components, lib); `packages/ui` grows the Radix-based primitives the admin screens need (Dialog, AlertDialog, DropdownMenu, Select, Slider, Switch, Progress, Table, Badge, Empty, Pagination, Combobox); `src/admin/app.tsx` → `app.ts` (accessGate + API routes + shell serving + redirects; pages/islands/ui deleted); new read routes assemble from the existing `src/admin-data.ts`, `src/health.ts`, `src/usage.ts`, `src/insights.ts`, `src/normalize-admin.ts`, `src/reconcile-admin.ts`, `src/audit-admin.ts`, `src/ingest-db.ts`, `src/satellite-audit-db.ts`, `src/discovery-db.ts` readers (no new `env.DB` access); markdown rendering stays Worker-side (`src/admin/markdown.ts`).
- **Config/build**: worker `package.json` gains `"./admin-api"` types-only export, drops the retired devDeps and the `src/admin/client` typecheck pass; root/worker `build:admin` scripts repoint; new `aubr dev:admin` (Vite HMR proxying `/admin/api`); `wrangler.jsonc` untouched.
- **CI/deploy**: `ci.yml` test job builds the admin app via `aubr build:admin` (now Vite); `admin-ui` job unchanged in name and blocking-ness (its screenshot path filters gain `packages/admin-app/**` + `packages/ui/**`); `trigger-deploy` path filter gains `packages/admin-app/`; `data-deploy.yml` unchanged (it already runs `aubr build:admin`).
- **Docs (same pass)**: `src/admin/CLAUDE.md` (rewritten), root `CLAUDE.md` + `CONTRIBUTING.md` (admin panel description, build commands, packages), `docs/ARCHITECTURE.md` (admin surface), `README.md`. `docs/TOOLS.md`/`docs/SCHEMAS.md` untouched (no tool or data-shape change; no D1 migration).
- **Risk**: parity regressions (mitigated: the ported Playwright suite's existing assertions are the acceptance bar, normative parity checklist in design.md); the Access-cookie-expiry mid-session fetch failure (handled: detected in the shared fetch layer, surfaced as a reload prompt); serving mistakes shadowing `/admin/api` or the 404 guard (covered by kept + new passthrough specs).
- **Non-goals**: the design bundle's Data › Flyer deals browser (`FlyerScreen.jsx`) and the Nodes tab's pinned-edge mutations (`NodeEdgeDialog`) — both are designed-but-never-built surfaces with no backing routes; parity-first, deferred to follow-up changes. No offline/persistence layer (P5's posture is member-app-only), no member-app changes, no auth-machinery changes, no CORS.
