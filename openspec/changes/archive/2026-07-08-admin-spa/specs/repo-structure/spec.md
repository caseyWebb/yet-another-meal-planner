## MODIFIED Requirements

### Requirement: Code repository layout

The code repository (`caseyWebb/groceries-agent`) SHALL be an **`aube` workspace monorepo** (workspaces declared in `pnpm-workspace.yaml`, packages cross-depending via the `workspace:*` protocol) whose packages are the **Worker** (the Cloudflare Worker — MCP server, member `/api`, and the Access-gated `/admin` routes — with its build tooling `scripts/`, D1 migrations `migrations/d1/`, and Worker tests), a shared **contract** package, the home-network **satellite** package, the **member app** (`packages/app`, the React SPA served at `/`), the **admin app** (`packages/admin-app`, the operator panel SPA served at `/admin`), and the shared **UI** package (`packages/ui`, the shadcn/ui primitives and theme tokens both apps consume). The shared **contract** package SHALL be **workerd-pure** (no Node-only dependencies) because it is imported by BOTH the Worker (workerd runtime) and the satellite (Node runtime); it SHALL hold the recipe-parse spine (JSON-LD extraction + normalize) and the ingest wire types/validators, so the parse and the payload shape can never drift between the two runtimes. The **satellite** package (Node + a headless-browser dependency) SHALL be isolated in its own workspace so its heavy dependencies never enter the Worker build or deploy. The reference docs SHALL remain under `docs/`, the OpenSpec workspace under `openspec/`, and the agent persona source `AGENT_INSTRUCTIONS.md` at a defined location. Reusable CI workflows that operator data repos call SHALL live under `.github/workflows/`. The code repository SHALL NOT commit a generated plugin bundle, a `.claude-plugin/marketplace.json`, or any built static-assets bundle (the Worker's merged assets root — both SPAs' output — is a gitignored build artifact); those are generated by CI/the operator's deploy, not committed.

#### Scenario: The repository is a workspaces monorepo

- **WHEN** the code repository is cloned and listed
- **THEN** it resolves as a workspaces monorepo with a Worker package, a shared workerd-pure contract package, a satellite package, the member-app, admin-app, and shared-UI packages, alongside `docs/`, `openspec/`, and `AGENT_INSTRUCTIONS.md`, and no committed plugin bundle, `.claude-plugin/marketplace.json`, or built assets bundle

#### Scenario: The shared contract package is workerd-pure

- **WHEN** the contract package is built
- **THEN** it carries no Node-only dependency, so both the Worker (workerd) and the satellite (Node) import the same recipe-parse and ingest wire types

#### Scenario: The satellite's heavy deps stay out of the Worker

- **WHEN** the Worker is built or deployed
- **THEN** the satellite package's browser/Node dependencies are not included — they are isolated to the satellite workspace
