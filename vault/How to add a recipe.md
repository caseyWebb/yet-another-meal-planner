# How to add a recipe

This vault is the **authoring** surface for the shared grocery-agent recipe corpus.
You write recipes here; **Remotely Save** syncs them to the operator's Cloudflare **R2**
bucket; the Worker's scheduled reconcile validates and indexes them. Friends never open
this vault — they read recipes through the agent and the cookbook site.

> [!info] One-time setup (first open only)
> 1. **Trust the plugins.** Obsidian opens new vaults in *Restricted Mode*. Open
>    **Settings → Community plugins → Turn on community plugins** (trust this vault).
>    Metadata Menu, Templater, and Remotely Save are bundled and pinned.
> 2. **Enter your R2 sync credentials.** Open **Settings → Remotely Save**, choose the
>    **S3-compatible** remote, and paste the endpoint, bucket, region, and your own
>    scoped access key + secret (the operator issues you a per-author R2 token — it is
>    **not** shipped in this vault). Run **Remotely Save: start sync** once to confirm.

## Adding a recipe

1. **Create a note in `recipes/`.** Templater applies the **New recipe** template
   automatically — a frontmatter skeleton plus `## Ingredients` / `## Instructions`.
2. **Fill the facets from the Properties dropdowns.** The `recipe` fileClass binds the
   controlled-vocabulary fields to dropdowns generated from the server's own vocabulary
   (`src/vocab.js`):
   - **protein** and **cuisine** — pick exactly one (or leave empty for none).
   - **season** and **requires_equipment** — pick zero or more.
   - **course** — pick one or more (e.g. `main`, `side`); this list is open, so you may
     add a value the suggestions don't cover.

   Because these dropdowns come from the same source the server validator uses, an
   off-vocabulary value such as `poltry` is **not offered** — you get the fast, in-editor
   feedback that constrained authoring needs.
3. **Write the body.** Keep the `## Ingredients` and `## Instructions` headings — the
   cookbook renders them.
4. **Save.** Remotely Save pushes the file to R2; the next reconcile tick indexes it (or
   records a fixable error if something is still off).

## What you do *not* author

- **`description`** — generated by the Worker from your facets and stored separately;
  the template has no field for it on purpose. Don't add one; it would be ignored.

> [!warning] The dropdowns are a convenience, not the gate
> The constrained dropdowns give instant feedback, but the **server-side reconcile is
> authoritative**. If you edit frontmatter by hand (or with plugins off), the reconcile
> still validates it — a constrained dropdown never lets invalid content skip that
> backstop.
